### Phase 1: 기반 설정 및 데이터 전처리 모듈 개발

2. 데이터 전처리 클래스(core/preprocess.py) 초기 구현. DICOM 로드, 16비트 정규화, CLAHE 기반 대비 개선 파이프라인 구축.
3. 학습에 활용 가능한 외부 오픈 데이터셋(Mendeley, TUFTS) 조사 및 README.md에 다운로드 링크 정리 완료.
4. 다음 작업: core/model.py 내 SwinIR-Light 아키텍처 구현 및 학습 루프 기본 틀 작성 예정.

### 데이터셋 정보 갱신 및 Tufts 인용 정보 반영

1. 기존 Mendeley 데이터셋 링크 만료 확인에 따른 정보 갱신.
2. 대체 데이터셋으로 Kaggle(Mendeley 미러) 및 Hugging Face(DENTEX) 데이터셋 조사 완료 및 README.md 반영.
3. Tufts Dental Database 사용을 위한 공식 인용문(Citation) 정보를 README.md에 추가.

### Phase 2: SwinIR-Lightweight 모델 및 데이터셋 구현

1. SwinIR-Lightweight 아키텍처(core/model.py) 구현 완료. M2 Pro 16GB RAM 환경을 고려하여 4개의 RSTB 블록과 PixelShuffle 업샘플러를 사용하는 경량화 구조로 설계.
2. Tufts Dental Database 전용 데이터셋 클래(core/dataset.py) 구현 완료. 고해상도(HR) 이미지 크롭 및 저해상도(LR) 합성(노이즈 및 다운샘플링) 파이프라인 구축.
3. MPS 디바이스 연동 테스트를 통해 M2 Pro GPU에서의 기본 연산 흐름 확인.

### Phase 3: 학습 엔진 구축 및 본격 가동

1. MPS 가속 기반 학습 스크립트(scripts/train_mps.py) 작성 완료. L1 Loss와 Adam 옵티마이저를 활용한 SR 학습 루프 구성.
2. 멀티프로세싱 시 OpenCV 객체 피클링 에러(CLAHE) 및 넘파이 음수 스트라이드(negative strides) 이슈 해결.
3. M2 Pro 최고 성능 모드에서 Tufts 데이터셋 기반 학습 개시. 첫 에폭 평균 손실 약 0.082 수준으로 안정적 출발 확인.

### Phase 4: 학습 완료 및 성능 평가

1. 총 100에폭 학습 완료. 평균 L1 Loss가 초기 0.082에서 최종 0.017대까지 하락하며 안정적으로 수렴함.
2. 최종 모델 체크포인트(checkpoints/pano_swinir_epoch_100.pth) 저장 완료.
3. 추론 스크립트(scripts/inference.py)를 통한 가시적 성능 평가 수행. Input 노이즈가 효과적으로 제거되고 치아 경계선이 선명하게 복원됨을 확인.

##### 2026-02-23 (3일차) #####

2. 통합 데이터셋 클래스(PanoDataset) 구현 완료. Tufts(JPG)와 DENTEX(PNG)의 서로 다른 디렉토리 구조 및 파일 포맷을 통합 지원하도록 개선.
3. config/base_config.yaml 설정을 업데이트하여 여러 데이터 소스(root_dirs)를 리스트 형태로 관리할 수 있도록 구조 변경.
4. train_mps.py 및 inference.py의 데이터 로딩 로직을 PanoDataset 기반으로 갱신 완료.
5. 테스트 결과, 통합 데이터셋에서 총 4,142개의 학습용 이미지가 정상적으로 로드됨을 확인.

### 대규모 재학습 개시 및 타일링 시스템 구현

1. 약 4,100장의 통합 데이터를 기반으로 고정밀 재학습(Fine-tuning) 가동. MPS 가속 및 최고 성능 모드를 통해 에폭당 약 90초 내외로 안정적 진행 중.
2. 타일링 및 심리스 블렌딩 모듈(core/tiling.py) 구현 완료. 4K 이상의 거대 파노라마 이미지를 중첩 타일로 분할 처리하고, 코사인 윈도우 마스크를 이용해 경계면 없이 합성하는 로직 구축.
3. 대용량 이미지 처리 시 발생할 수 있는 메모리 문제를 방지하기 위해 타일 단위 순차 처리를 기본값으로 설정.
