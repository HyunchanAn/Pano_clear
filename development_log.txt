### 이슈 1~4번 동시 해결 및 코드베이스 통합
- [Issue 1] MPS 하드웨어 종속성 탈피 및 Cross-Platform 가속 엔진 적용
  - `src/pano_clear/device.py` 내 `get_onnx_execution_providers()` 함수를 추가하여 TensorRT/CUDA, CoreML, CPU 등 구동 환경에 따른 최적 실행 공급자(Execution Provider) 분기 로직 추상화 완료.
  - `scripts/export_onnx.py`를 점검하여 ONNX 포맷 변환 파이프라인 정상 연동 확인.
  - `train_mps.py` 및 `upscale_sample.py`에서 하드코딩된 디바이스 할당 로직을 `get_best_device()`로 일괄 교체.

- [Issue 2] Tiling 윈도우 오버랩 영역의 연산 중복 및 GPU I/O 병목 개선
  - `src/pano_clear/tiling.py` 리팩토링. 
  - `process_large_image` 함수 내에서 타일들을 `torch.stack`으로 단일 배치화하여 모델에 전달하도록 Batch Tiling 방식 도입.
  - `merge_tiles` 함수에서 불필요한 블렌딩 대신 고정 크기 마진을 계산하여 중복 연산 없이 유효 영역만 크롭 및 결합하는 슬라이딩 윈도우 방식으로 전환하여 연산 속도 최적화.

- [Issue 3] 16-bit DICOM 스케일 미지정 및 해상도 클리핑 데이터 손실 위험 방지
  - `src/pano_clear/preprocess.py` 내에 실제 비트 심도가 12-bit/16-bit인지 검증하는 `assert` 로직 유지.
  - DICOM 원본 메타데이터(Window Center/Width)가 없을 시 치아 경계부를 뭉개지 않도록 1~99 백분위수 기반 정규화를 수행하는 로직 검증 완료.

- [Issue 4] Iterative SR (x4~x16) 연산 시 에러 전파 및 환각 현상 검증 장치 추가
  - `scripts/evaluate_metrics.py`를 확장하여 x2 단계 뿐만 아니라 반복 호출 시(x4, x8, x16) 단계별로 다운샘플링 원본 대비 PSNR/SSIM 저하 추세를 추적할 수 있도록 벤치마크 코드 전면 개편.
  - 평가 중 SSIM이 0.75 미만으로 떨어질 시 환각 현상 가능성에 대한 경고(Artifact Warning) 문구 출력 기능 통합.

- `pyproject.toml`에 pytest용 `pythonpath = ["src"]` 설정을 추가하여 테스트 환경 복구.
- streamlit cloud 배포 테스트 진행하지 않음 - 다음 작업 때 제일 먼저 해야 함.

1. 프로젝트 개요 파악: SwinIR-Lightweight 기반의 치과 파노라마 영상 화질 개선 프로젝트임을 확인. M2 Pro 환경에 최적화된 타일링 시스템과 MPS 가속이 핵심.
2. 학습 상태 확인: 4,600장 규모의 대규모 데이터셋으로 100에폭 학습이 완료되었으며, 최종 체크포인트(checkpoints/pano_swinir_epoch_100.pth)가 정상적으로 존재함을 확인.
3. 서비스 인터페이스 확인: Streamlit 기반의 app.py 구현 완료 확인. Cloud 환경 배포를 고려하여 CPU 연산 모드 및 메모리 최적화 로직이 적용되어 있음.
4. 주요 모듈 검토:
   - core/model.py: RSTB 기반 SwinIR-Light 아키텍처 구현.
   - core/tiling.py: 중첩 코사인 윈도우 블렌딩을 통한 대용량 이미지 처리 로직.
   - core/preprocess.py: DICOM 및 16비트 이미지 전처리 파이프라인.
5. 다음 단계 제안 준비: 현재 모델 성능 고도화나 실제 배포 환경에서의 사용자 인터페이스 개선, 또는 추가적인 후처리 로직(예: 선명도 강조) 도입 가능성 검토.

### UI 및 기능 대규모 개편 및 고도화 완료

1. 텐서 크기 불일치 오류(Tensor Mismatch) 해결: tiling.py에 reflect padding 로직과 타일 클리핑(clipping) 로직을 추가하여 다양한 크기의 이미지 입력 시 발생하는 OOB(Out of Bounds) 에러를 원천 차단.
2. 실전 모드 도입: 데모용 인위적 노이즈를 제거하고, 사용자가 업로드한 이미지를 원본 해상도 그대로 강화하는 '실전 모드'를 기본값으로 설정.
3. 단계별 다중 배율 지원(Iterative Upscaling): 모델의 기본 x2 배율을 기반으로, x4, x8, x16까지 확장할 수 있는 반복 추론 루프 구현.
4. 타임라인(History) 뷰 도입: st.session_state를 활용하여 추론된 이미지가 초기화되지 않고 아래로 누적되며 시각화되는 인터랙티브 UI 구축. 개별 이미지 다운로드 지원.
5. 실시간 샤프닝(Medical Sharpening): 치근 및 피질골 경계선 시인성 확보를 위해, 원본 데이터 손실 없이 화면 표시 및 저장 시에만 적용되는 실시간 언샤프 마스킹 필터 추가 및 슬라이더 연동.
